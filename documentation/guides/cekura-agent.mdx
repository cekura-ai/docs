## Overview

Cekura Agent is an AI assistant built into your dashboard. Describe what you want in plain language and the agent handles it — no need to navigate menus or fill out forms.

The agent has full access to your workspace and can:
- Create and configure AI agents
- Build evaluators and test scenarios
- Define custom metrics
- Run tests and view results
- Manage call logs and observability data
- Help with first-time setup and onboarding

## Getting Started

Open Cekura Agent from the navigation menu in your dashboard. Click **Cekura Agent** to open the chat interface.

Type your request in the message box and press Enter. The agent responds immediately and takes action on your behalf.

**Example requests:**

- "Create a Retell agent named Support Bot"
- "Show me my test results from the last 24 hours"
- "Generate 5 test scenarios for my booking agent"
- "What metrics should I use for a customer service agent?"

## How It Works

The agent connects to Cekura's API using your authenticated session. No additional API key setup is required.

When you send a message:
1. The agent acknowledges your request
2. It calls the appropriate API operations
3. Results appear in the chat

For complex requests, the agent may ask clarifying questions before proceeding.

## Chat Sessions

Maintain multiple conversations with different contexts:

- **New chat** — Click the plus icon to start a fresh conversation
- **Switch chats** — Click any previous chat in the sidebar to continue it
- **Delete chat** — Click the trash icon to remove a conversation

Chat history persists across browser sessions.

## Capabilities

### Creating Resources

Ask the agent to create agents, evaluators, metrics, and other resources:

- "Create a VAPI agent for my dental practice assistant"
- "Add a metric that checks if the agent confirms the appointment time"
- "Create 10 evaluators that test different booking scenarios"

### Running Tests

Trigger test runs without navigating to the evaluators page:

- "Run all evaluators for my Support Bot agent"
- "Run the appointment booking evaluator 3 times"
- "Start a load test with 5 concurrent calls"

### Viewing Data

Query your workspace data conversationally:

- "Show me failed calls from today"
- "List my agents"
- "What was the success rate for last week's tests?"

### Getting Help

Ask questions about Cekura features and best practices:

- "How do I set up Retell integration?"
- "What's the difference between binary and continuous metrics?"
- "How do I configure webhooks?"

## Stopping a Request

If the agent is processing a long-running operation, click the stop button to cancel. The agent halts and shows any partial results.

## Tips

- **Be specific** — Include names, IDs, or numbers when referring to existing resources
- **One task at a time** — The agent handles requests sequentially; wait for completion before sending the next message
- **Check results** — After the agent creates something, verify it in the dashboard if needed
