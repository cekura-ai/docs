## What You'll Learn

How to use Cekura's Auto Improve feature to analyze test failures and automatically refine your agent's system prompt.

## The Problem

After running tests, you'll find evaluators where your agent fails — wrong responses, missed steps, incorrect tool calls. Manually analyzing each failure and rewriting your prompt is time-consuming and error-prone.

## How Auto Improve Works

Auto Improve analyzes your test results — specifically the failures — and generates a revised prompt that addresses the identified issues.

<Steps>
  <Step title="Run Your Tests">
    Run a batch of evaluators against your agent. The more tests you run, the better Auto Improve understands the failure patterns.
  </Step>

  <Step title="Review Results">
    In the results view, identify evaluators that failed. Look at the transcripts to understand what went wrong.
  </Step>

  <Step title="Trigger Auto Improve">
    Select the run and click **Improve Prompt**. Cekura analyzes:

    - Which evaluators passed and which failed
    - The conversation transcripts
    - Your current agent description
    - The expected outcomes vs. actual outcomes
  </Step>

  <Step title="Review Suggested Changes">
    Auto Improve presents a diff of your current prompt vs. the suggested revision. Review the changes to ensure they make sense for your use case.
  </Step>

  <Step title="Apply and Re-test">
    Apply the improved prompt and re-run your tests to verify the improvements. Iterate as needed.
  </Step>
</Steps>

## Best Practices

<CardGroup cols={2}>
  <Card title="Run Enough Tests" icon="chart-line">
    Run at least 10-20 evaluators before using Auto Improve. More data = better improvements.
  </Card>
  <Card title="Review Before Applying" icon="magnifying-glass">
    Always review the suggested changes. Auto Improve is a suggestion, not a blind override.
  </Card>
  <Card title="Iterate" icon="arrows-rotate">
    Run tests → improve → re-test → improve again. Each cycle catches more edge cases.
  </Card>
  <Card title="Track Progress" icon="chart-mixed">
    Compare pass rates across iterations to measure improvement.
  </Card>
</CardGroup>

<Tip>
Auto Improve works best when your evaluators have clear, specific expected outcomes. Vague outcomes make it harder for the system to identify what needs to change.
</Tip>
