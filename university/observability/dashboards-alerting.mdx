## What You'll Learn

How to create custom dashboards with widgets to visualize trends, and set up alerting to catch issues early.

## Dashboards

Dashboards are containers for **widgets** — individual charts that visualize a specific field from your call data.

### Create a Dashboard

<Steps>
  <Step title="Navigate to Dashboards">
    Go to the **Dashboards** section in the sidebar.
  </Step>
  <Step title="Create New Dashboard">
    Click **Create Dashboard** and give it a name (e.g., "Production Health", "Weekly Metrics").
  </Step>
  <Step title="Add Dashboard Filters (Optional)">
    Set filters that apply to all widgets in the dashboard — e.g., only show calls from the last 7 days or for a specific agent.
  </Step>
</Steps>

### Add Widgets

Each widget plots one field as a chart.

<Steps>
  <Step title="Click Add Widget">
    In your dashboard, click **Add Widget**.
  </Step>
  <Step title="Choose Field">
    Select what to visualize:

    | Field | Chart Types | Example |
    |-------|------------|---------|
    | `duration` | Line, Bar | Call duration trends |
    | `success` | Pie, Bar | Pass/fail distribution |
    | `call_ended_reason` | Pie, Bar | Why calls end |
    | `metric_evaluations.value` | Line, Bar, Pie | Metric score trends |
    | `metadata.*` | Pie, Bar | Custom metadata distribution |
  </Step>
  <Step title="Configure Chart">
    - **Chart type**: Line (trends), Bar (aggregated), or Pie (distribution)
    - **Aggregation**: Count, Sum, Avg, Min, Max
    - **Time period**: Hour, Day, Week, or Month
    - **Widget filters**: Additional filters for this specific widget
  </Step>
</Steps>

### Filter Reference

Filters use JSON conditions:

```json
{
  "operator": "and",
  "conditions": [
    { "field": "timestamp", "op": "gte", "value": "today-7d" },
    { "field": "success", "op": "eq", "value": true }
  ]
}
```

For metric-specific filters, use `same_row` to ensure conditions match the same evaluation:

```json
{
  "operator": "same_row",
  "conditions": [
    { "field": "metric_evaluations.metric.name", "op": "eq", "value": "CSAT" },
    { "field": "metric_evaluations.value", "op": "gte", "value": 4 }
  ]
}
```

## Alerting

Combine dashboards with alerting to get notified when metrics degrade:

- **Low Credits Alerts** — email when credits drop below threshold ([set up here](/documentation/advanced/credit-alerts))
- **Webhook Alerts** — trigger webhooks based on call outcomes ([webhook format](/documentation/advanced/webhook-format))
- **Cron Job Monitoring** — schedule evaluator runs and get notified on failures

<Tip>
Create a "Critical Metrics" dashboard that tracks your most important metrics (expected outcome pass rate, latency P95, CSAT average) and review it daily. Combine with cron jobs to catch regressions early.
</Tip>
