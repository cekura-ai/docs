## Field Filtering with the `ql` Parameter

Use the `ql` query parameter to control which fields are returned in the response.

### Default Behavior

By default, large fields (`transcript`, `transcript_object`, and `evaluation`) are excluded from the response for better performance.

### Include All Fields

To include all fields including transcript and evaluation data, use `ql={*}`:

<CodeGroup>

```bash Curl
curl -X GET "https://api.cekura.ai/observability/v1/call-logs/?agent_id=123&ql={*}" \
  -H "X-CEKURA-API-KEY: <your-api-key>"
```

```python Python
import requests

def list_calls_with_all_fields(api_key, agent_id):
    url = 'https://api.cekura.ai/observability/v1/call-logs/'
    headers = {
        'X-CEKURA-API-KEY': api_key
    }
    params = {
        'agent_id': agent_id,
        'ql': '{*}'
    }

    response = requests.get(url, headers=headers, params=params)
    return response.json()
```

```javascript JavaScript
async function listCallsWithAllFields(apiKey, agentId) {
  const url = new URL('https://api.cekura.ai/observability/v1/call-logs/');
  url.searchParams.append('agent_id', agentId);
  url.searchParams.append('ql', '{*}');

  const response = await fetch(url, {
    method: 'GET',
    headers: {
      'X-CEKURA-API-KEY': apiKey
    }
  });

  return await response.json();
}
```

</CodeGroup>

### Select Specific Fields

To request only specific fields, list them inside the curly braces:

<CodeGroup>

```bash Curl
curl -X GET "https://api.cekura.ai/observability/v1/call-logs/?agent_id=123&ql={id,call_id,success,timestamp}" \
  -H "X-CEKURA-API-KEY: <your-api-key>"
```

```python Python
import requests

def list_calls_selective(api_key, agent_id):
    url = 'https://api.cekura.ai/observability/v1/call-logs/'
    headers = {
        'X-CEKURA-API-KEY': api_key
    }
    params = {
        'agent_id': agent_id,
        'ql': '{id,call_id,success,timestamp}'
    }

    response = requests.get(url, headers=headers, params=params)
    return response.json()
```

```javascript JavaScript

```

</CodeGroup>

This returns only the `id`, `call_id`, `success`, and `timestamp` fields.

### Exclude Specific Fields

To exclude specific fields, prefix field names with a minus sign (`-`):

<CodeGroup>

```bash Curl
curl -X GET "https://api.cekura.ai/observability/v1/call-logs/?agent_id=123&ql={-duration,-metadata}" \
  -H "X-CEKURA-API-KEY: <your-api-key>"
```

```python Python
import requests

    url = 'https://api.cekura.ai/observability/v1/call-logs/'

    params = {
        'agent_id': agent_id,

```

```javascript JavaScript

  const url = new URL('https://api.cekura.ai/observability/v1/call-logs/');
```

</CodeGroup>

This returns all default fields except `duration` and `metadata`.
