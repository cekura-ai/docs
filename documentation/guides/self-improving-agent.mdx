## What is the Self-Improving Agent?

The Self-Improving Agent analyzes evaluation failures and production call issues to improve your AI voice agent's prompts. It runs an automated loop that diagnoses problems, proposes fixes, applies changes, and validates the results.

It's designed for teams who want to iterate on prompts faster without manually reviewing transcripts and rewriting instructions.

<Note>
  The Self-Improving Agent is available through the **cekura-self-improving-agent** skill in Claude Code and other AI coding agents. See [Cekura Skills](/mcp/skills) for installation instructions.
</Note>

## How it works

The Self-Improving Agent runs a multi-phase optimization loop:

<Steps>
  <Step title="Diagnose">
    The agent analyzes evaluation results, failed runs, or production call logs to identify patterns in what went wrong. It categorizes issues by severity and frequency.
  </Step>
  <Step title="Propose">
    Based on the diagnosis, the agent drafts specific prompt changes to address the identified issues. You can review these changes before they're applied.
  </Step>
  <Step title="Apply">
    The proposed changes are applied to your agent's prompt configuration.
  </Step>
  <Step title="Validate">
    The agent re-runs evaluations to verify the changes improved performance. If issues remain, the loop continues with another iteration.
  </Step>
</Steps>

The loop runs for up to 10 iterations or until the agent determines no further improvements can be made.

## Input sources

You can seed the Self-Improving Agent with evaluation signals from multiple sources:

| Source | Description |
|--------|-------------|
| **Scenarios** | Re-run specific evaluator scenarios to generate fresh results |
| **Result batch** | Use an existing result batch as the starting point |
| **Completed runs** | Analyze specific completed test runs |
| **Production calls** | Improve prompts based on real production call logs |

## Auto mode vs. manual approval

The Self-Improving Agent supports two modes:

- **Auto mode (default)**: The agent applies prompt changes automatically after each iteration without waiting for approval.
- **Manual approval mode**: The agent pauses after the propose phase and waits for you to approve or reject the changes before applying them.

Use manual approval mode when you want to review each change before it affects your agent.

## Using the skill

The Self-Improving Agent is accessed through the `cekura-self-improving-agent` skill. After installing [Cekura Skills](/mcp/skills), the skill activates when you describe a task like:

- "Improve my agent based on the last test run"
- "Auto-tune agent 42 from production calls"
- "Fix the issues found in result batch 15"

The skill guides you through selecting the input source and monitoring progress as the optimization loop runs.

## Rate limits

Self-improving agent runs are rate-limited to **1 request per minute per user** to manage computational resources.

## Best practices

- **Start with a focused set of failures.** 10-20 representative failed runs produce better results than thousands of calls.
- **Review changes in manual mode first.** Until you trust the optimization quality, use manual approval to see what changes are being proposed.
- **Combine with Metric Lab.** The Self-Improving Agent optimizes prompts; [Metric Lab](/documentation/guides/metric-lab) optimizes metrics. Use both for comprehensive improvement.
- **Re-run evaluations after optimization.** Validate that improvements generalize beyond the seed data.
