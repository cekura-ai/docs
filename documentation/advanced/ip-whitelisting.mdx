If you need to whitelist Cekura's IP addresses for infrastructure access, you should use the static IP addresses provided below.

## Production Environment

The following IP is used for outbound requests from Cekura production services:

- **32.184.152.201**

<Note>
All outbound traffic from Cekura production egresses through a single NAT gateway, so only this one IP needs to be whitelisted.
</Note>

## Use Cases

You should whitelist this IP when:

- Providing presigned URLs for audio files or recordings
- Exposing webhook endpoints with IP restrictions
- Configuring firewall rules for API integrations
- Setting up private API access

## SIP Calls

SIP calls from Cekura are routed through **Twilio's SIP infrastructure**, not through the NAT gateway IP above. If your SIP endpoint or SIP trunk uses IP-based access control (ACLs), you must whitelist **Twilio's SIP signaling and media IPs** rather than `32.184.152.201`.

See [Twilio's SIP trunking IP addresses](https://www.twilio.com/docs/sip-trunking/ip-addresses) for the current list of IPs to whitelist.

## Region

This IP is designated for the **US-WEST-2 region** where Cekura production services are hosted.
