## What You'll Learn

By the end of this lesson you'll have a fully configured agent on Cekura, ready for testing or observability.

<Note>
An "Agent" in Cekura represents **your** AI voice or chat agent — the one you're testing or monitoring. It is not the evaluator that simulates callers.
</Note>

## Step 1: Create Your Agent

<Steps>
  <Step title="Navigate to Agent Creation">
    Go to **Agent Settings** from the left navigation bar. Click **Create an Agent**.
  </Step>

  <Step title="Fill in Basic Details">
    - **Agent Name**: A descriptive name (e.g., "Customer Support Bot", "Scheduling Assistant")
    - **Language**: The primary language your agent speaks
  </Step>

  <Step title="Add Your Agent Description">
    Paste your agent's **system prompt** into the description field. This enables Cekura to generate accurate evaluators automatically.

    <Tip>
    If your agent has multiple states or nodes, you can structure the description as JSON. Even a single large block of text works — Cekura can process it.
    </Tip>

    For provider-specific exports:
    - **Retell**: Agents → Select agent → Export (downloads `.json`)
    - **Vapi**: Workflows → Select agent → Code button → Copy
  </Step>
</Steps>

## Step 2: Configure Your Provider

Select your voice/chat provider and enter the required credentials.

| Provider | Required Fields |
|----------|----------------|
| **Retell** | API Key, Assistant ID |
| **VAPI** | Private API Key, Assistant ID |
| **ElevenLabs** | API Key, Agent ID |
| **LiveKit** | API Key, API Secret, URL, Agent Name |
| **Pipecat** | Cloud API Key, Agent Name |
| **Custom** | Webhook URL |

<Note>
The **provider API key** is the most important configuration. It enables provider-specific features like auto-sync, richer call data, and outbound calling. Always provide it if possible.
</Note>

## Step 3: Set Up Your Connection

Choose how Cekura connects to your agent:

<CardGroup cols={3}>
  <Card title="Phone (PSTN)" icon="phone">
    Add your agent's phone number. Cekura calls it directly. Simplest setup.
  </Card>
  <Card title="WebRTC / SDK" icon="signal">
    Connect via LiveKit, Pipecat, or ElevenLabs WebSocket for lower latency and richer metadata.
  </Card>
  <Card title="Chat / WebSocket" icon="comment">
    Use provider chat integrations or a custom WebSocket for text-based testing — 10x faster and 90% cheaper than voice.
  </Card>
</CardGroup>
