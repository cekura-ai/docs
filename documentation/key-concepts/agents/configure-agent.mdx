## Select Your Provider

Cekura supports a wide range of voice and chat providers. Select the provider that matches your agent's infrastructure to enable testing and observability.

<Info>
Don't see your provider? You can use a [Custom Integration](/documentation/integrations/custom-integration) to connect any platform via webhooks, or contact [support@cekura.ai](mailto:support@cekura.ai).
</Info>

## Integration Settings

Go to **Agent Settings** and configure the integration for your provider. Each provider requires different credentials and identifiers.

<AccordionGroup>
  <Accordion title="Retell" icon="/logo/retell_icon1.svg">
    **Required fields:**
    - **Retell API Key**
    - **Retell Assistant ID**

    Retell supports voice testing via phone numbers and chat testing via a chat agent copy.

    **Learn more:** [Retell Testing](/documentation/integrations/retell/testing) | [Retell Observability](/documentation/integrations/retell/observability)
  </Accordion>

  <Accordion title="VAPI" icon="/logo/vapi_icon1.svg">
    **Required fields:**
    - **VAPI Private API Key**
    - **VAPI Assistant ID**

    VAPI supports voice testing via phone numbers and chat testing via the VAPI chat integration.

    **Learn more:** [VAPI Testing](/documentation/integrations/vapi/testing) | [VAPI Observability](/documentation/integrations/vapi/observability)
  </Accordion>

  <Accordion title="ElevenLabs" icon="/logo/elevenlabs_icon1.svg">
    **Required fields:**
    - **ElevenLabs API Key**
    - **ElevenLabs Agent ID**

    ElevenLabs supports voice testing via WebSocket or phone numbers, and chat testing via the ElevenLabs chat integration.

    **Learn more:** [ElevenLabs Testing](/documentation/integrations/elevenlabs/testing) | [ElevenLabs Observability](/documentation/integrations/elevenlabs/observability)
  </Accordion>

  <Accordion title="LiveKit" icon="/logo/livekit_icon1.svg">
    **Required fields:**
    - **LiveKit API Key**
    - **LiveKit API Secret**
    - **LiveKit URL** (e.g., `wss://your-server.livekit.cloud`)
    - **Agent Name** (the specific agent name to dispatch)

    **Optional fields:**
    - **LiveKit Config (JSON)** — additional room configuration parameters, accessible via `ctx.room.metadata` in your agent

    **Learn more:** [LiveKit Testing](/documentation/integrations/livekit/testing) | [LiveKit Tracing](/documentation/integrations/livekit/tracing) | [LiveKit Manual Testing](/documentation/integrations/livekit-integration)
  </Accordion>

  <Accordion title="Pipecat" icon="/logo/pipecat_icon1.svg">
    **Required fields:**
    - **Pipecat Cloud API Key**
    - **Pipecat Agent Name**

    **Optional fields:**
    - **Assistant ID**
    - **Agent Configuration (JSON)**
    - **Room Properties (JSON)** — Daily.co room configuration

    **Learn more:** [Pipecat Automated Testing](/documentation/integrations/pipecat/automated) | [Pipecat Manual Testing](/documentation/integrations/pipecat/manual)
  </Accordion>

  <Accordion title="Custom" icon="plug">
    For providers not listed above, use the custom integration to send call transcripts via webhooks.

    **Required:** Your system sends POST requests to the Cekura webhook URL with call data.

    **Learn more:** [Custom Integration](/documentation/integrations/custom-integration)
  </Accordion>
</AccordionGroup>

## Connections

After selecting your provider, configure how Cekura connects to your agent for testing.

### Voice Connections

<Tabs>
  <Tab title="Phone Number (PSTN)">
    The simplest setup: add your agent's phone number in the **Contact Number** field. Cekura calls your agent directly via the phone network.

    ![Phone number configuration](/images/Phonenumber.png)

    This is the default for inbound voice testing. Your voice provider (Retell, VAPI, ElevenLabs, etc.) handles routing the call to your agent.
  </Tab>

  <Tab title="WebRTC / Provider SDK">
    Connect directly to your agent via WebRTC, bypassing the phone network entirely. This provides lower latency and richer metadata.

    Go to **Voice Integration** -> Choose your **Provider** -> Fill in the required fields listed above.

    <img src="/images/livekit/livekit-voice-integration-config.png" alt="Voice integration configuration" />

    Supported providers for WebRTC testing:
    - [LiveKit](/documentation/integrations/livekit/testing) — automated room and token management
    - [Pipecat](/documentation/integrations/pipecat/automated) — Pipecat Cloud integration
    - [ElevenLabs](/documentation/integrations/elevenlabs/testing) — WebSocket-based connection
  </Tab>

  <Tab title="SIP">
    For agents with SIP endpoints, Cekura sends SIP INVITE requests directly to your agent's SIP URI. This provides enhanced control, reduced latency, and custom header injection.

    Go to **Agent Settings** -> Configure your **SIP Endpoint** (e.g., `sip:agent@yourdomain.com`).

    Optional: Add digest authentication credentials if your SIP endpoint requires them.

    See [SIP Testing](/documentation/integrations/sip-integration) for full setup details.
  </Tab>
</Tabs>

### Chat Connections

<Tabs>
  <Tab title="Provider Chat Integration">
    Many voice providers also support chat/text-based testing. This is the fastest way to iterate — 10x faster and 90% cheaper than voice.

    Go to **Chatbot Integration** -> Choose your **Provider** -> Enter your **Assistant ID**.

    ![No-code chat integration](/images/nocodeintegration.png)

    Supported providers:
    - [Retell](/documentation/integrations/retell/testing) — requires a chat agent copy
    - [VAPI](/documentation/integrations/vapi/testing) — uses your existing Assistant ID
    - [ElevenLabs](/documentation/integrations/elevenlabs/testing) — uses your existing Agent ID
    - [Bland](/documentation/integrations/chat-testing) — uses a Pathway ID

    See [Chat-Based Testing](/documentation/integrations/chat-testing) for the full guide.
  </Tab>

  <Tab title="Custom WebSocket">
    Connect your own WebSocket server for chat testing. This gives you full control over the conversation handling.

    Go to **Chatbot Integration** -> Choose **Custom** -> Enter your **WebSocket URL** (starting with `wss://`).

    ![Custom WebSocket configuration](/images/customchatbot.png)

    See the [Chat-Based Testing guide](/documentation/integrations/chat-testing) for WebSocket message format, authentication headers, and a Python server example.
  </Tab>

  <Tab title="SMS / WhatsApp">
    For SMS-based agents, add your agent's **Phone Number** in the Contact Number field. Cekura sends text messages to test your SMS flows.

    ![Phone number configuration](/images/phonenumber.png)

    WhatsApp-based testing follows the same pattern if your provider supports it.
  </Tab>
</Tabs>
