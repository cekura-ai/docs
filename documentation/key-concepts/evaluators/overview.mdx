## What are Evaluators?

Evaluators are test cases for your AI voice agents. Each evaluator simulates a conversation with your agent to systematically test its performance and behavior.

<Note>
  Evaluators are the test scenarios that need to be simulated. Metrics define
  what needs to be tested and how success is measured.
</Note>

## Evaluator Setup Guide

There are two methods of creating evaluators:

- [Automatic](#automatic)
- [Manual](#manual)

### Automatic

![](/images/generate-evaluator.png)

<Steps>
  <Step title="Click Generate Evaluator">
    From the Evaluators section, click the <b>Generate Evaluator</b> button on
    the top right.
  </Step>
  <Step title="Select Number of Evaluators (Required)">
    At minimum, select the number of evaluators to generate (1-100). This is the
    only required field.
<Note>

  <b>Quick tip:</b> You can simply enter the number of evaluators and skip directly
  to Step 4 if you want to use default settings!
</Note>
<b>Example:</b> 5 evaluators
  </Step>
  <Step title="Additional Configuration (Optional)">
    <b>Optional: Additional Configuration Parameters</b> (found in the <i>
  Additional Info
</i> dropdown):

    <ul>
      <li>
        <b>Test Profile:</b> Select or create a test profile to give evaluators
        identity (e.g., name, date of birth).
      </li>
      <li>
        <b>Knowledge Base Files:</b> Choose KB files to inform evaluator
        generation for more context-relevant scenarios.
      </li>
      <li>
        <b>Personalities:</b> Pick AI personalities to test various interaction
        styles.
      </li>
      <li>
        <b>Extra Instructions:</b> Add further evaluator generation instructions
        (e.g., "Generate evaluators with frustrated customers about delayed
        orders").
      </li>
      <li>
        <b>Generate Expected Outcomes:</b> Toggle to auto-generate expected
        outcomes for each evaluator.
      </li>
      <li>
        <b>Tags:</b> Add tags to organize and categorize evaluators ("Press
        Enter" after each tag).
      </li>
      <li>
        <b>Additional Configuration:</b> Access advanced options like phone
        numbers, first message, and tool configurations by expanding this
        section.
      </li>
    </ul>

  </Step>
  <Step title="Click Generate and Wait">
    Review your configuration and click <b>Generate</b>. Once generated, you may
    expand the evaluators to review details.
    <Note>

      Evaluator generation may take some time. You can close the popup after clicking
      generate and explore the rest of the platform.
    </Note>

  </Step>
</Steps>

### Manual

![Create Evaluator](/images/create-evaluator.png)

<Steps>
  <Step title="Click Create Evaluator">
    Go to the **Evaluators** section and click the <b>Create Evaluator</b> button on the top right.
  </Step>
  <Step title="Fill in Mandatory Fields">
    Complete the required fields to create your evaluator:

    <b>Name</b>: Descriptive name (e.g. "Order Return - Incorrect Order ID")

    <b>Personality</b>: Select desired tone and behavioral style ([see details](/documentation/key-concepts/evaluators/personality))

    <b>Instructions</b>: Define evaluator behavior ([see details](/documentation/key-concepts/evaluators/instructions))

  </Step>
  <Step title="Additional Configuration (Optional)">
    Optionally add extra settings:

    <b>Tags</b>: Organize evaluators. Press <b>Enter</b> after each tag (e.g., "returns", "verification").

    <b>Test Profile</b>: Assign identity info (name, DOB, etc.), or use the default if none selected. [More info](/documentation/key-concepts/evaluators/test-profile)

    <b>Language</b>: Set evaluator's conversation language.

    <b>Expected Outcome Prompt</b>: Define successful conversation criteria. [More info](/documentation/key-concepts/evaluators/expected-outcome)

    <b>Metrics</b>: Select additional metrics to evaluate. eg: Latency, Relevancy, Consistency, etc. [More info](/documentation/key-concepts/metrics/overview)

    <b>Phone Number</b>: Assign an optional phone number.

    <b>First Message</b>: Optionally specify the evaluatorâ€™s starting message.

    <b>Tools</b>: Enable tools like <b>dtmf</b> (tone dialing), <b>end_call</b>, or <b>end_call_only_on_transfer</b>.

  </Step>
  <Step title="Create Evaluator">
    Review all your settings and click the <b>Create</b> button to save your evaluator.
    <Note>
      You can edit any of these fields later by selecting the evaluator and clicking Edit.
    </Note>
  </Step>
</Steps>
