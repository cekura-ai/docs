## What is Multi-Turn Red Teaming?

Multi-turn red teaming uses AI-driven attack strategies that adapt across conversation turns. Unlike standard red teaming, which pulls single-turn prompts from a pre-built database, multi-turn attacks simulate persistent adversaries who refine their approach based on your agent's responses.

<Note>
  Multi-turn red teaming generates 5-10 conversation turns per scenario, with the attack strategy evolving based on how your agent responds.
</Note>

## Attack Categories

Multi-turn red teaming supports six attack categories:

<CardGroup cols={2}>
  <Card title="System Prompt Leak" icon="file-shield" color="#EF4444">
    Attempts to extract the system prompt, internal instructions, or configuration details from your agent
  </Card>
  <Card title="Data Leak" icon="database" color="#F97316">
    Tries to extract sensitive business data, customer PII, or confidential information your agent has access to
  </Card>
  <Card title="Harmful Content" icon="triangle-exclamation" color="#EAB308">
    Attempts to make your agent generate harmful content, misinformation, or inappropriate responses
  </Card>
  <Card title="Unauthorized Actions" icon="ban" color="#22C55E">
    Tests for safety bypasses, excessive agency, or unauthorized commitments your agent might make
  </Card>
  <Card title="Off-Task" icon="shuffle" color="#3B82F6">
    Attempts to hijack the conversation or lead your agent into off-topic behavior
  </Card>
  <Card title="Jailbreak" icon="unlock" color="#8B5CF6">
    General attempts to bypass safety guardrails and make your agent ignore its instructions
  </Card>
</CardGroup>

## How to Generate Multi-Turn Scenarios

<Steps>
  <Step title="Navigate to Your Agent">
    Go to the agent for which you want to create multi-turn red-teaming scenarios.
  </Step>
  <Step title="Open the Evaluator Tab">
    Click on the **Evaluator** tab and then click **Generate Evaluators**.
  </Step>
  <Step title="Select Multi-Turn Red-Teaming">
    In the dialog box that appears:
    - Set the **number of scenarios** to generate
    - Select **Red-Teaming** as the scenario type
    - Enable the **Multi-Turn** option
  </Step>
  <Step title="Choose Attack Category">
    Select which attack category you want to test against:
    - System Prompt Leak
    - Data Leak
    - Harmful Content
    - Unauthorized Actions
    - Off-Task
    - Jailbreak
  </Step>
  <Step title="Add Extra Instructions (Optional)">
    Provide additional context to guide the attack strategy generation. For example:
    - Specific data you want to protect (for Data Leak tests)
    - Business rules you want to enforce (for Unauthorized Actions tests)
    - Topics that should be off-limits (for Off-Task tests)
  </Step>
  <Step title="Generate and Run">
    Click **Generate** and then run the generated scenarios to test your agent.
  </Step>
</Steps>

## Multi-Turn vs Standard Red Teaming

<AccordionGroup>
  <Accordion title="Standard Red Teaming" icon="bolt">
    - Uses pre-built attack prompts from a database of 32,617+ scenarios
    - Single-turn attacks that don't adapt
    - Broader coverage across 150 categories
    - Faster generation and execution
    - Best for: Comprehensive baseline security testing
  </Accordion>
  <Accordion title="Multi-Turn Red Teaming" icon="arrows-spin">
    - AI-generated attack strategies that evolve across turns
    - 5-10 turns per conversation with adaptive behavior
    - Focused on six high-impact attack categories
    - Simulates persistent, sophisticated attackers
    - Best for: Testing against advanced, persistent threats
  </Accordion>
</AccordionGroup>

## When to Use Multi-Turn Red Teaming

Use multi-turn red teaming to:

- **Protect sensitive data**: Test whether attackers can extract confidential information through sustained conversation
- **Prevent prompt leaks**: Verify your system prompt stays protected against sophisticated extraction attempts
- **Enforce boundaries**: Confirm your agent maintains appropriate limits under persistent pressure
- **Simulate real attacks**: Model how actual adversaries probe your agent across multiple interactions

<Tip>
  For comprehensive security testing, we recommend running both standard red teaming (for broad coverage) and multi-turn red teaming (for depth against sophisticated attacks).
</Tip>
