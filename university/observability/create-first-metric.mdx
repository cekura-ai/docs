## What You'll Learn

How to create a custom metric that uses an LLM to evaluate your agent's calls based on natural language criteria.

## What are Metrics?

Metrics are judges that evaluate call quality. They analyze transcripts (and sometimes recordings) to score calls on specific criteria. Cekura offers two types:

1. **Pre-defined metrics** — CSAT, latency, interruption count, etc. Available out of the box.
2. **Custom metrics** — You define the criteria in natural language (LLM Judge) or Python code.

## Pre-Defined Metrics

Start by enabling pre-defined metrics for instant insights:

Go to **Metrics** and add the built-in options. These require no configuration and cover common needs like latency, call quality, and customer satisfaction.

## Create a Custom LLM Judge Metric

<Steps>
  <Step title="Navigate to Metrics">
    Go to your agent's **Metrics** section and click **Create Metric**.
  </Step>

  <Step title="Name Your Metric">
    Give it a descriptive name: e.g., `Appointment Booked`, `Correct Greeting`, or `Policy Followed`.
  </Step>

  <Step title="Choose Metric Type">
    Select the output type:

    | Type | Use Case | Output |
    |------|----------|--------|
    | **Boolean** | Pass/fail checks | True / False |
    | **Rating** | Quality scores | 0-5 scale |
    | **Enum** | Categories | Custom labels (e.g., "Excellent", "Poor") |
  </Step>

  <Step title="Write the Description">
    This is your metric's evaluation prompt. Write in natural language what constitutes success.

    **Example:**
    ```
    Check if the Main Agent successfully booked an appointment for
    the caller. The appointment must include a confirmed date,
    time, and the agent must have repeated the details back to
    the caller for confirmation.
    ```

    <Tip>
    Use **context variables** to make metrics dynamic:
    - `{{transcript}}` — the full conversation
    - `{{agent_description}}` — your agent's system prompt
    - `{{metadata.instructions}}` — custom metadata from the call
    </Tip>
  </Step>

  <Step title="Test Before Saving">
    Click **Test Metric**, select a few past Call IDs, and run the test. Verify the metric passes/fails as expected on historical data.
  </Step>

  <Step title="Create Metric">
    If results look good, click **Create Metric** to save.
  </Step>
</Steps>

## Affects Call Success

Toggle **Affects Call Success** to `True` if this metric is critical — meaning if it fails, the entire call is considered a failure. This is useful for must-pass criteria like identity verification or appointment confirmation.

## Project-Level vs. Agent-Level

- **Project-level metrics** are shared across all agents in the project
- **Agent-level metrics** apply to a specific agent only

Enable or disable project-level metrics per agent as needed.
