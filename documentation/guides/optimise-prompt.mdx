## What is Optimise Prompt?

**Optimise Prompt** is a button that hands the selected scenarios to
[Cekura Agent](/documentation/guides/cekura-agent) and starts an automated
self-improvement loop on the underlying agent. Cekura Agent diagnoses why the
scenarios are failing, proposes targeted edits to the agent's prompt (and
configuration where applicable), applies them, redeploys, and re-runs
the scenarios — repeating until the agent reaches a 100% pass rate on the
validation set or a stop condition fires.

It closes the loop between *"my scenarios are red"* and *"my agent is
fixed"* without you having to read every failing transcript, write the edits
yourself, or re-trigger the runs by hand.

## Where to find it

The button is labeled **Optimise Prompt** (with a Sparkles icon and a
**BETA** badge). It appears in two places:

### Scenarios page

In the Scenarios toolbar, alongside **Run** and the delete control. The
button is enabled whenever at least one scenario is selected — the selected
scenarios become the validation set for the loop.

### Result page

In the header of any individual run's result page, next to **Share Report**.
Clicking it picks up every scenario that ran in the result plus the result
ID, so the loop seeds itself from the runs you just looked at.

## When to use it

Use Optimise Prompt when:

- You have an agent connected to Cekura and at least one scenario configured against it.
- Recent runs of those scenarios are failing (or have been reviewed as failures).
- The failures look like prompt-following or configuration issues — not infrastructure outages, missing API keys, or upstream data problems.

If most of your failures are voice-quality, latency, or infrastructure issues,
fix those first — the loop will skip them and may exit early with an upstream
hand-off.

## What happens when you click it

Clicking the button:

1. Collects the scenario IDs from your current selection (or every scenario that ran in the result).
2. Navigates to **Cekura Agent** (`/ai-assistant`) with the scenario IDs — and the result ID, when launched from a result page — encoded in the URL.
3. Cekura Agent auto-creates a new chat and sends a `/self-improve` command with those IDs as its first message. No extra prompt or configuration step is required.
4. The chat then runs the **self-improving-agent** skill, announcing each phase as it goes.

There is no pre-flight drawer or input form — selection on the Scenarios /
Result page IS the input. To narrow scope, narrow your selection before
clicking.
