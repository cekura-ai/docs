The Results page shows how your agent performed after an evaluator run — pass/fail breakdown, failure patterns, latency metrics, and AI-generated recommendations for what to fix next.

## Results Page Layout

The Results page displays four main sections at the top:

1. **Success Rate** — A pie chart showing the overall pass/fail ratio across all runs.
2. **What went wrong** — A breakdown of failure categories (failed to connect, infrastructure issues, workflow issues, successful calls).
3. **Recommended next steps** — AI-generated recommendations for fixing identified issues.

Below these summary cards, you'll find:

- **Failure breakdown** — A detailed table of failure categories with drill-down capability.
- **Metrics** — Performance and quality metrics organized by type.
- **Runs table** — Individual run results with transcripts and evaluations.

## Cekura AI Summary

When your result contains failures, Cekura generates an AI summary that analyzes patterns across failing runs. The summary appears in the **Failure breakdown** section and includes:

- **What happened** — A concise description of the primary failure patterns.
- **Why it happened** — Analysis of root causes behind the failures.

The summary generates automatically after the result completes. For recent results, you may see a "Generating summary…" indicator while the analysis runs.

<Note>
The AI summary analyzes patterns across multiple failing runs to identify systemic issues rather than one-off errors. It works best when you have several runs with similar failure modes.
</Note>

### Regenerating the Summary

If you've made changes to your agent and want a fresh analysis, or if the summary seems outdated, click **Regenerate** to queue a new summary generation. The page will automatically update when the new summary is ready.

## Recommended Next Steps

The **Recommended next steps** card displays AI-generated action items based on the failure analysis. Each recommendation includes:

- A numbered priority order
- A title describing the action
- A brief description of what to change

These recommendations are designed to address the most impactful issues first. Use them as a starting point for your agent improvements.

## Failure Breakdown Table

The failure breakdown table provides a structured view of workflow failures across your runs:

| Column | Description |
|--------|-------------|
| **Workflow failures** | The failure category or issue type, color-coded by severity (red for rank 1, orange for rank 2, yellow for lower priority). |
| **In plain English** | A human-readable explanation of what went wrong. |
| **Runs affected** | The specific run IDs that experienced this failure. |

### Drilling into Failures

Click any row in the failure breakdown table to open a detailed view of the affected runs. This lets you:

- Review individual transcripts
- See metric evaluations for each run
- Understand the specific context of each failure

### Ask About Failures

The **Ask about failures** button opens an interactive chat where you can ask questions about the failure patterns. Use this when:

- You need more context about why failures occurred
- You want to explore specific failure scenarios in detail
- The summary doesn't fully explain a particular issue

## Metrics Section

The Metrics section displays all evaluated metrics organized into collapsible groups:

### Rubric Metrics

Metrics included in your evaluator's rubric appear at the top. These show:

- Pass rate as a percentage
- Color-coded progress bars (green for high scores, yellow for medium, red for low)
- Click any metric to see which runs passed or failed

### Numeric Metrics

Quantitative measurements like latency and ringing duration:

- **Mean Latency** — Average response time in milliseconds, with P50, P95, and P99 percentiles.
- **Avg Ringing Duration** — For voice calls, the average time before the call connects.

### Other Metric Types

Additional sections may appear based on your configured metrics:

- **Boolean** — Pass/fail metrics
- **Rating** — Score-based qualitative assessments
- **Enum** — Categorical classifications
- **Tags** — Success rates grouped by scenario tags

Click any metric card to open a detailed view showing the evaluation for each run.

## Using Results to Improve Your Agent

1. **Start with the AI summary** — Get a quick read on the primary failure patterns.
2. **Review recommended next steps** — Prioritize the suggested fixes.
3. **Drill into specific failures** — Click through the failure breakdown to understand individual cases.
4. **Check metric performance** — Look for patterns across failing metrics.
5. **Use Ask about failures** — Get more context when the summary doesn't fully explain an issue.
6. **Iterate and re-run** — After making changes, run the same evaluators again to measure improvement.
