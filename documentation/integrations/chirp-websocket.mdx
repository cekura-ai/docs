## Overview

Test your voice agents directly over WebSocket connections using CHIRP (Conversational Handoff for Inter-agent Realtime Protocol). Cekura connects to your WebSocket endpoint and exchanges raw PCM audio in real-time, enabling low-latency voice testing without phone infrastructure.

## When to Use CHIRP WebSocket

Use CHIRP WebSocket testing when:

- Your agent does not have a phone number
- You already have a real-time audio pipeline over WebSocket
- You need bidirectional streaming with lower latency than telephony
- Your system handles raw PCM audio natively

For agents with phone numbers, consider [SIP testing](/documentation/integrations/sip-integration) or standard voice testing instead.

## How It Works

1. **Connection**: Cekura opens a WebSocket connection to your configured `wss://` endpoint
2. **Authentication**: Optional HTTP Basic Auth credentials are sent during the WebSocket handshake
3. **Audio Exchange**: Raw PCM audio is exchanged as binary WebSocket frames in both directions
4. **Evaluation**: Cekura captures audio, generates transcripts, and evaluates metrics

### Audio Format

| Property | Value |
|----------|-------|
| Encoding | PCM signed 16-bit little-endian |
| Sample rate | 16,000 Hz |
| Channels | Mono (1 channel) |
| Transport | Binary WebSocket frames |

## Prerequisites

Configure the CHIRP WebSocket endpoint via the Agent API or dashboard:

**Required field:**
- **chirp_websocket_url**: The `wss://` endpoint where your agent receives audio connections

**Optional fields:**
- **chirp_basic_auth_username**: HTTP Basic Auth username
- **chirp_basic_auth_password**: HTTP Basic Auth password

### Agent Configuration Example

```bash
curl -X PATCH \
  'https://api.cekura.ai/test_framework/v1/aiagents/{agent_id}/' \
  -H 'X-CEKURA-API-KEY: <YOUR_API_KEY>' \
  -H 'Content-Type: application/json' \
  -d '{
    "chirp_data": {
      "chirp_websocket_url": "wss://your-host.com/voice",
      "chirp_basic_auth_username": "user",
      "chirp_basic_auth_password": "secret"
    }
  }'
```

## API Endpoint

```
POST https://api.cekura.ai/test_framework/v1/scenarios/run_scenarios_chirp/
```

## Authentication

Include your API key in the request header:
```
X-CEKURA-API-KEY: <YOUR_API_KEY>
```

## Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `scenarios` | array | Yes* | List of scenario IDs to run |
| `tags` | array | Yes* | Filter scenarios by tags |
| `folder_path` | string | Yes* | Filter scenarios by folder |
| `agent_id` | integer | No | Target agent ID; inferred from scenarios if omitted |
| `frequency` | integer | No | Number of times to run each scenario (default: 1) |
| `personality_ids` | array | No | Personality IDs to apply |
| `test_profile_ids` | array | No | Test profile IDs to apply |
| `concurrency_limit` | integer | No | Maximum concurrent runs |
| `name` | string | No | Custom name for this result set |

*One of `scenarios`, `tags`, or `folder_path` is required.

## Examples

### Single Run (cURL)

```bash
curl -X POST \
  'https://api.cekura.ai/test_framework/v1/scenarios/run_scenarios_chirp/' \
  -H 'X-CEKURA-API-KEY: <YOUR_API_KEY>' \
  -H 'Content-Type: application/json' \

    "scenarios": [30]
  }'
```

### Multiple Runs (cURL)

```bash
curl -X POST \
  'https://api.cekura.ai/test_framework/v1/scenarios/run_scenarios_chirp/' \
```

## CHIRP Protocol Reference

CHIRP is a transport-only protocol for exchanging raw audio over WebSocket. For the full protocol specification, including message formats and event handling, see the [CHIRP protocol documentation](https://docs.getbluejay.ai/simulation-integrations/websockets).

### Key Protocol Details

- **Binary frames**: Raw PCM audio (16kHz, 16-bit, mono)
- **Text frames (optional)**: JSON events for speech detection (`speech.started`, `speech.completed`)
- **Authentication**: HTTP Basic Auth during WebSocket upgrade
- **Graceful close**: WebSocket close code `1000`

<Tip>
A minimal CHIRP server only needs to accept the WebSocket connection, validate authentication, and exchange binary audio frames. Text events are optional.
</Tip>
