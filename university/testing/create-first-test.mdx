## What You'll Learn

How to create an evaluator (test case), run it against your agent, and interpret the results.

## What is an Evaluator?

An evaluator is a test case that simulates a conversation with your agent. Each evaluator has:

- **Instructions** — what the simulated caller should do
- **Expected Outcome** — what success looks like
- **Metrics** — measurements like latency, relevancy, and consistency
- **Personality** — language, tone, and behavioral characteristics (optional)
- **Test Profile** — identity information like name and DOB (optional)

## Create Your First Evaluator

<Steps>
  <Step title="Navigate to Evaluators">
    Select your agent in the dashboard, then go to the **Evaluators** tab.
  </Step>

  <Step title="Click Create Evaluator">
    Click **Create New Evaluator** to open the creation form.
  </Step>

  <Step title="Write Instructions">
    Describe what the simulated caller should do. Be specific.

    **Example:**
    ```
    Call to cancel an appointment scheduled for next Tuesday.
    Provide your date of birth when asked for verification.
    ```
  </Step>

  <Step title="Set Expected Outcome">
    Define what a successful conversation looks like.

    **Example:**
    ```
    The agent successfully cancels the appointment and provides
    a confirmation number.
    ```
  </Step>

  <Step title="Add Metrics">
    Select the metrics to evaluate. For your first test, enable all **pre-defined metrics** for comprehensive analysis:

    - Latency
    - Infrastructure Issues
    - Relevancy
    - Consistency
    - CSAT
  </Step>

  <Step title="Save">
    Click **Save** to create your evaluator.
  </Step>
</Steps>

## Run Your Test

<Steps>
  <Step title="Select Evaluators">
    In the Evaluators tab, check the boxes next to the evaluators you want to run.
  </Step>
  <Step title="Click Run">
    Click the **Run** button. Choose the number of iterations if you want to run the same test multiple times.
  </Step>
  <Step title="Wait for Results">
    Cekura handles voice synthesis, transcript generation, and conversation management. Results typically appear within a few minutes.
  </Step>
</Steps>

## Read Your Results

Each test run produces a result with:

| Field | What It Shows |
|-------|---------------|
| **Expected Outcome** | Met or Not Met — did the agent do what it was supposed to? |
| **Transcript** | Full conversation between evaluator and your agent |
| **Metrics** | Scores for each attached metric (latency, relevancy, etc.) |
| **Pass/Fail** | Overall status based on expected outcome and critical metrics |

<Tip>
Click into any result to see the full transcript and metric breakdown. Use this to identify exactly where your agent succeeded or failed.
</Tip>
