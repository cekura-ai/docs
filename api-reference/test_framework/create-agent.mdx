## Provider configuration

Set `provider.type` to your hosting platform. Each provider requires different credentials inside the `provider` block — the tabs below show the exact fields for each.

<Tabs>
  <Tab title="Self-hosted">
    Use `self_hosted` when your agent is already deployed on your own infrastructure with a dedicated phone number. Cekura observes and evaluates calls on that number without needing provider credentials.

    **Fields**

    | Field | Type | Required | Description |
    |---|---|---|---|
    | `provider.type` | string | ✅ | `"self_hosted"` |
    | `provider.agent_id` | string | — | Optional identifier for your agent. |
    | `provider.credentials.api_key` | string | — | Shared secret Cekura includes in webhook headers for verification (write-only). |
    | `provider.send_post_conversation_metadata` | boolean | — | When `true`, Cekura expects your platform to POST call metadata to the observe endpoint after each call ends. |
    | `provider.chat_agent_details.type` | string | — | Set to `"self_hosted"` to enable text-mode test runs via WebSocket. |
    | `provider.chat_agent_details.config.url` | string | — | WebSocket endpoint (`wss://`) to connect for text-mode runs. |
    | `provider.chat_agent_details.config.headers` | object | — | Headers to include on the WebSocket upgrade request (e.g. `{"Authorization": "Bearer token"}`). |

    ```json
    {
      "name": "Support Bot",
      "description": "Handles inbound support calls",
      "project": 1376,
      "language": "en",
      "provider": {
        "type": "self_hosted",
        "credentials": {
          "api_key": "shared_secret_for_webhook_verification"
        },
        "send_post_conversation_metadata": true
      },
      "telephony": {
        "phone_number": "+14155551234",
        "inbound": true
      }
    }
    ```

    To also enable text-mode test runs:

    ```json
    {
      "name": "Support Bot (text-mode)",
      "project": 1376,
      "language": "en",
      "provider": {
        "type": "self_hosted",
        "chat_agent_details": {
          "type": "self_hosted",
          "config": {
            "url": "wss://staging.example.com/agent",
            "headers": { "Authorization": "Bearer token_xxx" }
          }
        }
      },
      "telephony": {
        "phone_number": "+14155551234",
        "inbound": true
      }
    }
    ```
  </Tab>

  <Tab title="VAPI">
    Connect a VAPI assistant. Supply your VAPI API key — Cekura validates it on create. The `agent_id` is the VAPI assistant ID and is optional; omit it to let Cekura pick up calls by phone number.

    **Fields**

    | Field | Type | Required | Description |
    |---|---|---|---|
    | `provider.type` | string | ✅ | `"vapi"` |
    | `provider.credentials.api_key` | string | ✅ | Your VAPI private API key (write-only). |
    | `provider.agent_id` | string | — | VAPI assistant ID (e.g. `asst_abc123`). |
    | `provider.credentials.config.public_key` | string | — | VAPI public key for web/SDK calls. |
    | `provider.auto_dial_outbound` | boolean | — | Auto-place outbound calls when running outbound scenarios. |
    | `provider.auto_import_calls` | boolean | — | Auto-import calls from VAPI every 30s into Cekura's call log. |
    | `provider.auto_sync_prompt` | boolean | — | Auto-sync the assistant's system prompt from VAPI every 30s. |
    | `provider.chat_agent_details.type` | string | — | Text-mode channel: `"vapi"`, `"sms"`, or `"whatsapp"`. |

    ```json
    {
      "name": "VAPI Sales Agent",
      "description": "Outbound sales qualification calls",
      "project": 1376,
      "language": "en",
      "provider": {
        "type": "vapi",
        "agent_id": "asst_abc123",
        "credentials": {
          "api_key": "vapi_sk_xxx",
          "config": {
            "public_key": "vapi_pk_xxx"
          }
        },
        "auto_dial_outbound": true,
        "auto_import_calls": true,
        "auto_sync_prompt": true
      },
      "telephony": {
        "phone_number": "+14155551234",
        "inbound": false
      }
    }
    ```
  </Tab>

  <Tab title="Retell">
    Connect a Retell agent. Both `credentials.api_key` (Retell API key) and `agent_id` (Retell agent ID) are required for voice runs. For text-mode runs, add `chat_agent_details` with a separate Retell chat agent ID.

    **Fields**

    | Field | Type | Required | Description |
    |---|---|---|---|
    | `provider.type` | string | ✅ | `"retell"` |
    | `provider.credentials.api_key` | string | ✅ | Your Retell API key (write-only). |
    | `provider.agent_id` | string | — | Retell agent ID for voice runs. |
    | `provider.credentials.config.trigger_url` | string | — | Retell trigger URL override (e.g. `https://api.retellai.com/create-phone-call`). |
    | `provider.auto_dial_outbound` | boolean | — | Auto-place outbound calls when running outbound scenarios. |
    | `provider.auto_import_calls` | boolean | — | Auto-import calls from Retell every 30s. |
    | `provider.auto_sync_prompt` | boolean | — | Auto-sync the agent's prompt from Retell every 30s. |
    | `provider.chat_agent_details.type` | string | — | Text-mode channel: `"retell"`, `"sms"`, or `"whatsapp"`. |
    | `provider.chat_agent_details.config.agent_id` | string | — | Retell chat agent ID (separate from the voice agent ID). |

## Telephony configuration

The `telephony` block configures the phone and SIP channels for the agent.

| Field | Type | Description |
|---|---|---|
| `telephony.phone_number` | string | E.164 phone number for inbound calls, e.g. `+14155551234`. |
| `telephony.inbound` | boolean | `true` = agent receives inbound calls; `false` = outbound only. |
| `telephony.sip_uri` | string | SIP URI for SIP-based integrations, e.g. `sip:user@domain.com`. |
| `telephony.sip_auth` | object | SIP credentials: `{"username": "...", "password": "..."}`. |
| `telephony.outbound_numbers` | array | Phone numbers authorized for outbound calls (E.164). Used for webhook validation — not required for calls to be placed. |

## Integration settings reference

These `provider`-level booleans control automatic background sync and dial behaviour. All default to `false`.

| Field | Supported providers | Description |
|---|---|---|
| `provider.auto_dial_outbound` | vapi, retell, elevenlabs, bland, livekit | Auto-place the outbound call to the scenario's configured number when running outbound evaluators. |
| `provider.auto_import_calls` | vapi, retell, elevenlabs | Poll the provider every 30s and import new calls into Cekura's call log. |
| `provider.auto_sync_prompt` | vapi, retell, elevenlabs, synthflow | Poll the provider every 30s and update the agent's system prompt in Cekura to match. |
| `provider.send_post_conversation_metadata` | self_hosted | Expect your platform to POST call metadata to the Cekura observe endpoint after each call ends. |
