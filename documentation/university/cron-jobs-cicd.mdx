## What You'll Learn

How to run tests automatically — on a schedule with cron jobs, or on every code change with GitHub Actions.

## Cron Jobs

Cron jobs run evaluators on a schedule and alert you if things fail. Use them for continuous monitoring of your agent's health.

### Create a Cron Job

<Steps>
  <Step title="Select Scenarios">
    Go to the **Evaluators** section, select the scenarios you want to automate, and click the **Action** button.
  </Step>
  <Step title="Choose Create Cron Job">
    Select **Create a Cron Job** from the dropdown.
  </Step>
  <Step title="Configure Schedule">
    - **Name**: Give it a descriptive name
    - **Cron Expression**: Set the schedule (e.g., `0 9 * * 1-5` for weekdays at 9 AM)
    - **Timezone**: Select your timezone

    Common schedules:
    | Expression | Meaning |
    |-----------|---------|
    | `0 0 * * *` | Daily at midnight |
    | `0 */6 * * *` | Every 6 hours |
    | `0 9 * * 1-5` | Weekdays at 9 AM |
    | `*/30 * * * *` | Every 30 minutes |
  </Step>
  <Step title="Save">
    Review and click **Save**. Your cron job is now active.
  </Step>
</Steps>

Manage existing cron jobs from **Project → Cron Jobs** in the top right corner.

## GitHub Actions CI/CD

Run your Cekura tests automatically on every push, pull request, or manual trigger.

### Quick Setup

<Steps>
  <Step title="Create Workflow File">
    Add `.github/workflows/test-agents.yml` to your repository:

    ```yaml
    name: Agent Tests
    on:
      push:
        branches: [main]
      pull_request:
        types: [opened, synchronize]

    jobs:
      run-tests:
        runs-on: ubuntu-latest
        steps:
          - name: Cekura Run Tests
            uses: cekura-ai/cekura-github-actions@v1.0.0
            with:
              agent_id: ${{ vars.AGENT_ID }}
              scenario_ids: ${{ vars.SCENARIO_IDS }}
              api_key: ${{ secrets.CEKURA_API_KEY }}
    ```
  </Step>
  <Step title="Add Secrets & Variables">
    In your repo: **Settings → Secrets and variables → Actions**

    - **Secret**: `CEKURA_API_KEY` — your Cekura API key
    - **Variables**: `AGENT_ID`, `SCENARIO_IDS` (comma-separated), optionally `TAGS`
  </Step>
  <Step title="Trigger a Run">
    Push a change or open a PR. The workflow runs automatically.
  </Step>
</Steps>

<Tip>
Use **tags** instead of scenario IDs for more flexibility. Tag your infrastructure tests with `infrastructure-suite` and run them as a group:

```yaml
tags: ${{ vars.TAGS }}   # e.g., "smoke-test,critical"
```
</Tip>
