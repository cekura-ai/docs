## What You'll Learn

The difference between inbound and outbound testing, when to use each, and how to set them up.

## Inbound Testing

Inbound is the default. Cekura's evaluator **calls your agent** — just like a real customer would.

**How it works:**
1. You provide your agent's phone number (or WebRTC/SIP endpoint)
2. Cekura initiates a call to your agent
3. The evaluator follows its instructions and converses with your agent
4. Results are evaluated after the call ends

**When to use:** Most testing scenarios — you want to test how your agent handles incoming calls.

**Setup:** Add your agent's contact number in **Agent Settings → Contact Number**, or configure a WebRTC/SIP connection.

## Outbound Testing

Outbound testing flips the direction — **your agent calls Cekura**. This tests agents that make outbound calls to customers (appointment reminders, follow-ups, sales calls).

**How it works:**
1. Cekura provides a phone number for your agent to call
2. You trigger your agent to make the outbound call (via API or manually)
3. The evaluator answers and follows its instructions
4. Results are evaluated after the call ends

### Auto-Call (Recommended)

For supported providers (Retell, VAPI, ElevenLabs), Cekura can trigger the outbound call automatically:

1. Set your agent to **outbound mode** in Agent Settings (toggle Inbound off)
2. Select evaluators and click **Run**
3. Cekura triggers your provider to make the call and handles everything

### Manual Outbound

For custom setups, use the API to orchestrate outbound tests programmatically. The flow:

1. Call the Run Evaluators API
2. Poll for pending runs — each returns a phone number
3. Trigger your agent to call that number
4. Monitor results

<Note>
For outbound testing, test profile fields are automatically sent to your agent as **dynamic variables** via the webhook payload. This mimics production where your system passes caller context when initiating calls.
</Note>

## Choosing Between Inbound and Outbound

| Feature | Inbound | Outbound |
|---------|---------|----------|
| Direction | Cekura → Your Agent | Your Agent → Cekura |
| Setup | Just add phone number | Configure outbound mode + trigger |
| Use case | Customer service, booking | Reminders, follow-ups, sales |
| Identity data | Evaluator speaks it verbally | Sent as dynamic variables |
