## What You'll Learn

How to send your production call data to Cekura so you can monitor, evaluate, and analyze real conversations.

## How Observability Works

Cekura's observability platform receives call transcripts and metadata from your production system, then automatically evaluates them against your metrics. This gives you continuous insight into how your agent performs with real users.

## Setup

<Steps>
  <Step title="Get Your API Key">
    Go to **Settings → API Keys** in the dashboard and copy your key.
  </Step>

  <Step title="Create an Agent">
    If you haven't already, create an agent in the dashboard. This agent represents the production agent you're monitoring.
  </Step>

  <Step title="Define Metrics">
    Go to your agent's **Metrics** section and add the metrics you want to evaluate production calls against. Start with pre-defined metrics (Latency, CSAT, etc.) and add custom ones later.
  </Step>

  <Step title="Start Pushing Logs">
    Use the Observe API to send call data after each production call ends.
  </Step>
</Steps>

## Sending Call Data

Send a POST request to the Observe API with your call transcript:

```bash
curl -X POST https://api.cekura.ai/observability/v1/observe \
  -H "X-CEKURA-API-KEY: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "call_id": "your-call-id",
    "transcript_type": "retell",
    "transcript_json": [...],
    "call_ended_reason": "customer_ended_call",
    "voice_recording_url": "https://...",
    "metadata": {
      "customer_id": "C-123",
      "region": "US"
    }
  }'
```

### Metadata

Include metadata in your observe requests to enable powerful filtering and analysis later. Common metadata fields:

- Customer ID, region, language
- Agent version or prompt version
- Call category or department
- Any business-specific context

<Tip>
Metadata fields become filterable in dashboards and can be used as dynamic variables in metric descriptions. Plan your metadata schema early.
</Tip>
