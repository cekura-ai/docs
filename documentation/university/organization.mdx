## What You'll Learn

How to organize your Cekura workspace so metrics, evaluators, and test profiles are reusable across agents — and your team can collaborate effectively.

## Projects: Your Top-Level Container

Everything in Cekura lives inside a **Project** — agents, evaluators, metrics, and results.

Key things to know:
- **Metrics are project-level** — shared across all agents in the project
- **RBAC is project-scoped** — Admin, Member, and Viewer roles per project
- **Results are filterable** — use the Views tab to create filtered views per agent

### How to Structure Projects

Ask yourself: **how do I maximize reusability of metrics and evaluators across agents?**

<AccordionGroup>
  <Accordion title="Teams & Small Orgs" icon="users" defaultOpen>
    One project per team works well when agents share common flows.

    ```
    Project: Healthcare Receptionist Agents
      Agents:
        - Clinic A Receptionist
        - Clinic B Receptionist
      (Shared metrics, evaluators, and test profiles)
    ```
  </Accordion>

  <Accordion title="Enterprise" icon="building">
    Layer team-level and environment-level separation.

    ```
    Project: Booking Team - Staging
    Project: Booking Team - Production
    Project: Outreach Team - Staging
    Project: Outreach Team - Production
    ```

    This gives you RBAC boundaries — devs access staging, leads access production.
  </Accordion>
</AccordionGroup>

## Organizing Evaluators

As your test suite grows, organize evaluators into **folders**.

<CardGroup cols={2}>
  <Card title="By Flow Type" icon="diagram-project">
    Group by workflow (Booking Flow, Identity Verification, Payment). Best when multiple agents share the same flows.
  </Card>
  <Card title="By Agent" icon="robot">
    Group by specific agent. Best when each agent has unique flows with little overlap.
  </Card>
</CardGroup>

<Tip>
Use **Actions → Duplicate** to copy evaluators across projects, and **Actions → Move** to reorganize into different folders.
</Tip>
