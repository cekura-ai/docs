## Overview

This guide walks you through setting up observability for your Pipecat-based voice agents using Cekura. Unlike other integrations that use a single webhook, Pipecat observability follows a two-step flow—first request temporary S3 credentials to upload your call audio, then submit the call data via webhook.

## Integration Flow

The Pipecat observability integration requires two API calls:

1. **Before or during the call**: Request scoped S3 credentials from Cekura to upload your audio recording
2. **After the call ends**: Submit the webhook payload with the transcript and audio reference

## Step 1: Get S3 Credentials for Audio Upload

Before or during a call, request temporary S3 credentials to upload your audio recording.

### Endpoint

```
GET https://api.cekura.ai/observability/v1/pipecat/egress-credentials?agent_id={agent_id}
```

### Authentication

Include your Cekura API key in the request header:

```
X-CEKURA-API-KEY: <your_cekura_api_key>
```

### Response

```json
{
  "access_key": "ASIA...",
  "secret_key": "...",
  "session_token": "...",
  "expires_at": "2026-04-18T12:00:00Z",
  "bucket": "cekura-recordings",
  "key": "pipecat_recordings/abc123.pcm",
  "region": "us-east-1"
}
```

### Upload Your Audio

Use the returned credentials to upload your audio file to S3. The audio must be in **raw PCM format** with the following specifications:

| Property | Value |
|----------|-------|
| Sample rate | 24 kHz |
| Channels | 2 (stereo) |
| Bit depth | 16-bit |

<Note>
The credentials are scoped to the specific S3 key returned in the response. You must upload to exactly that key. The credentials expire after 1 hour.
</Note>

## Step 2: Submit Call Data via Webhook

After the call ends, submit the call data to Cekura's observe endpoint.

### Endpoint

```
POST https://api.cekura.ai/observability/v1/pipecat/observe/
```

### Authentication

Include your Cekura API key in the request header:

```
X-CEKURA-API-KEY: <your_cekura_api_key>
```

### Request Body

```json
{
  "agent_id": 123,
  "session_id": "unique-session-id",
  "session_start": "2026-04-18T10:00:00Z",
  "session_end": "2026-04-18T10:05:30Z",
  "transcript": [
    {
      "started_at": "2026-04-18T10:00:01Z",
      "ended_at": "2026-04-18T10:00:05Z",
      "role": "assistant",
      "content": "Hello! How can I help you today?"
    },
    {
      "started_at": "2026-04-18T10:00:06Z",
      "ended_at": "2026-04-18T10:00:10Z",
      "role": "user",
      "content": "I'd like to check my appointment."
    }
  ],
  "audio_s3_key": "pipecat_recordings/abc123.pcm"
}
```

### Payload Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `agent_id` | integer | Yes | The ID of your agent in Cekura |
| `session_id` | string | Yes | Unique identifier for the call session |
| `session_start` | string | Yes | Call start time in ISO 8601 format |
| `session_end` | string | Yes | Call end time in ISO 8601 format |
| `transcript` | array | Yes | Array of transcript messages |
| `audio_s3_key` | string | Yes | The S3 key returned from the egress-credentials endpoint |

### Transcript Message Format

Each message in the transcript array should include:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `role` | string | Yes | Message role: "assistant", "user", or "tool" |
| `content` | string | Yes | The message content |
| `started_at` | string | No | Message start time in ISO 8601 format |
| `ended_at` | string | No | Message end time in ISO 8601 format |
| `tool_calls` | array | No | Tool call details (for assistant messages with tool calls) |
| `tool_call_id` | string | No | Tool call ID (for tool result messages) |

### Tool Call Format

For messages that include tool calls:

```json
{
  "role": "assistant",
  "tool_calls": [
    {
      "id": "call-uuid",
      "function": {
        "name": "get_appointments",
        "arguments": "{\"user_id\": \"123\"}"
      },
      "type": "function"
    }
  ]
}
```

For tool results:

```json
{
  "role": "tool",
  "content": "{\"appointments\": [...]}",
  "tool_call_id": "call-uuid"
}
```

### Response Codes

| Status | Description |
|--------|-------------|
| 200 | Call log created successfully |
| 400 | Invalid payload structure |
| 402 | Insufficient balance |
| 403 | Agent doesn't belong to API key's organization |
| 404 | Agent not found |
| 409 | Duplicate session_id (if duplicate call IDs are disabled) |

## Complete Example

Here's a complete Python example showing the full integration flow:

```python
import requests
import boto3
import io

CEKURA_API_KEY = "your_cekura_api_key"
CEKURA_AGENT_ID = 123
CEKURA_API_BASE_URL = "https://api.cekura.ai"

def get_egress_credentials(agent_id: int) -> dict:
    """Get temporary S3 credentials for uploading audio."""
    url = f"{CEKURA_API_BASE_URL}/observability/v1/pipecat/egress-credentials"
    headers = {"X-CEKURA-API-KEY": CEKURA_API_KEY}
    params = {"agent_id": agent_id}

    response = requests.get(url, headers=headers, params=params)
    response.raise_for_status()
    return response.json()


def upload_audio_to_s3(credentials: dict, pcm_data: bytes) -> None:
    """Upload PCM audio to S3 using the provided credentials."""
    s3_client = boto3.client(
        "s3",
        aws_access_key_id=credentials["access_key"],
        aws_secret_access_key=credentials["secret_key"],
        aws_session_token=credentials["session_token"],
        region_name=credentials["region"],
    )

    s3_client.put_object(
        Bucket=credentials["bucket"],
        Key=credentials["key"],
        Body=pcm_data,
    )


def send_call_to_cekura(
    session_id: str,
    session_start: str,
    session_end: str,
    transcript: list,
    audio_s3_key: str,
) -> dict:
    """Submit call data to Cekura's observe endpoint."""
    url = f"{CEKURA_API_BASE_URL}/observability/v1/pipecat/observe/"
    headers = {
        "X-CEKURA-API-KEY": CEKURA_API_KEY,
        "Content-Type": "application/json",
    }

    payload = {
        "agent_id": CEKURA_AGENT_ID,
        "session_id": session_id,
        "session_start": session_start,
        "session_end": session_end,
        "transcript": transcript,
        "audio_s3_key": audio_s3_key,
    }

    response = requests.post(url, headers=headers, json=payload)
    response.raise_for_status()
    return response.json()


# Example usage
def send_pipecat_call_to_cekura(
    session_id: str,
    session_start: str,
    session_end: str,
    transcript: list,
    pcm_audio_data: bytes,
):
    # Step 1: Get S3 credentials
    credentials = get_egress_credentials(CEKURA_AGENT_ID)

    # Step 2: Upload audio to S3
    upload_audio_to_s3(credentials, pcm_audio_data)

    # Step 3: Submit call data
    result = send_call_to_cekura(
        session_id=session_id,
        session_start=session_start,
        session_end=session_end,
        transcript=transcript,
        audio_s3_key=credentials["key"],
    )

    return result
```

## Important Considerations

1. **Audio format**: Audio must be raw PCM format (24 kHz, stereo, 16-bit). Other formats will not be processed correctly.

2. **Credential expiration**: S3 credentials expire after 1 hour. Request new credentials if the upload window has passed.

3. **Call duration limit**: Calls exceeding 60 minutes will be rejected.

4. **Session ID uniqueness**: Each `session_id` must be unique. Duplicate session IDs may be rejected depending on your agent's configuration.

5. **Upload timing**: Upload the audio file before submitting the webhook. The system verifies the file exists in S3 before processing.
