## Overview

Metric Alerts notify you when your AI agent's performance metrics need attention. Cekura supports two alert types:

- **Failure Alert**: Fires when a metric evaluation fails on a call
- **Drift Alert**: Fires when a metric's value shifts significantly from its historical baseline

## Alert Types

### Failure Alert

Failure Alerts trigger whenever a metric evaluation returns a failing result. Use these for binary metrics where any failure indicates an issue.

**When to use:** Quality checks, compliance requirements, or critical workflow steps that must pass on every call.

### Drift Alert

Drift Alerts use statistical analysis to detect when a metric's performance shifts from its historical baseline. Rather than alerting on every failure, they identify meaningful changes in patterns over time.

**When to use:** Monitoring trends in continuous metrics like satisfaction scores, latency, or talk ratio where occasional variation is expected but sustained shifts indicate problems.

## Creating an Alert

<Steps>
  <Step title="Navigate to Alerts">
    Go to **Settings** > **Alerts** in your project.
  </Step>

  <Step title="Add a New Alert">
    Click **Add Alert** and select the metric you want to monitor.
  </Step>

  <Step title="Choose Alert Type">
    Select either **Failure Alert** or **Drift Alert** based on your monitoring needs.
  </Step>

  <Step title="Configure Settings">
    For Failure Alerts, no additional configuration is needed.
    
    For Drift Alerts, choose a preset or customize the detection parameters.
  </Step>

  <Step title="Enable the Alert">
    Toggle the alert to **Enabled** and save your changes.
  </Step>
</Steps>

## Drift Alert Configuration

Drift Alerts detect statistically significant changes in metric performance. Use presets for common configurations or fine-tune the detection parameters.

### Presets

Cekura provides three presets that balance sensitivity and stability:

| Preset | Sensitivity | Reaction Speed | Sample Size | Best For |
|--------|-------------|----------------|-------------|----------|
| **Conservative** | Low (σ×3.0) | Stable (α=0.10) | 100 calls | Quiet metrics with infrequent changes |
| **Balanced** | Medium (σ×2.0) | Balanced (α=0.30) | 50 calls | General use — start here |
| **Aggressive** | High (σ×1.5) | Fast (α=0.50) | 25 calls | High-traffic metrics needing quick detection |

### Advanced Settings

Expand **Advanced settings** for fine-grained control:

**Sensitivity (std_multiplier)**
Controls how large a shift must be before the alert fires. Lower values mean higher sensitivity — the alert fires on smaller deviations from the baseline.

- σ×1.5: Very high sensitivity — fires on small shifts
- σ×2.0: High sensitivity
- σ×2.5: Medium sensitivity
- σ×3.0: Low sensitivity — only large shifts trigger alerts
- σ×4.0: Very low sensitivity

**Reaction Speed (ewma_alpha)**
Controls how quickly the baseline adapts to recent data. Higher values weight recent calls more heavily.

- α=0.05–0.15: Stable — baseline changes slowly, ignores short-term fluctuations
- α=0.20–0.35: Balanced — moderate adaptation to recent trends
- α=0.40–0.65: Fast — baseline reacts quickly to new data
- α=0.70–1.00: Very fast — heavily weights recent calls

**Sample Size (window_size)**
The number of recent calls used to calculate the baseline statistics. Larger samples provide more stable baselines but react more slowly to changes.

- Range: 10–500 calls
- Smaller values: Faster reaction, more volatile
- Larger values: More stable, slower to detect shifts

## Reviewing Alerts

When alerts fire, use the **Alert Review** page to understand what triggered them and take action.

### Accessing Alert Review

Click **Review Alerts** from the Alerts settings page, or navigate directly to the Alert Review section in your project.

### Review Page Features

The Alert Review page shows:

- **Total fires**: Number of times alerts fired in the selected time window
- **Unique alerts**: Count of distinct alerts that fired
- **Top firing alert**: The alert with the most fires
- **Activity timeline**: Chart showing when alerts fired over time
- **Alert list**: All alerts that fired, with fire counts and sparkline trends

Use the time range selector (24h, 3d, or 7d) to adjust the review window.

### Alert Detail View

Click any alert to see:

- **Fire events**: Individual alert fires with timestamps
- **Linked calls**: Direct links to the calls that triggered each fire
- **Activity chart**: Timeline of fires for this specific alert
- **Period comparison**: Change in fire count versus the prior period

### AI Summary

For alerts with multiple fires, click **Summarize with AI** to generate an analysis of the failures. The AI summary provides:

- **Headline**: Brief description of the issue pattern
- **Summary**: Overview of what's happening across the failures
- **Common themes**: Recurring patterns identified across the failing calls

This helps you quickly understand systemic issues without manually reviewing each call.

## Best Practices

- **Start with Balanced preset**: The Balanced preset works well for most metrics. Adjust only after observing alert behavior.
- **Use Failure Alerts for binary metrics**: When any failure matters, use Failure Alerts rather than trying to detect "drift" in pass/fail patterns.
- **Tune Drift Alerts based on traffic**: High-traffic metrics can use more aggressive settings since the sample sizes are naturally larger. Low-traffic metrics benefit from conservative settings to avoid false positives.
- **Review alerts regularly**: Use the Alert Review page to triage fires and identify patterns. The AI Summary feature helps surface root causes quickly.
- **Combine with Slack notifications**: Connect Slack to receive real-time notifications when alerts fire.

## Troubleshooting

**Alert fires too frequently**
- Increase the sensitivity multiplier (use Conservative preset or higher σ value)
- Increase the sample size to smooth out volatility
- Reduce the reaction speed (lower α value) to make the baseline more stable

**Alert doesn't fire when expected**
- Decrease the sensitivity multiplier (use Aggressive preset or lower σ value)
- Reduce the sample size for faster detection
- Increase the reaction speed (higher α value) to weight recent data more heavily

**Drift Alert shows "Custom" instead of a preset**
- Your current settings don't match any preset. This is normal if you've fine-tuned the parameters. The alert will still function correctly.
